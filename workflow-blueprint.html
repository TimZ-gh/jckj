Alignment method

本次对齐方式

我们不会让各部门从头编写流程，而是先提供一版业务理解稿。各部门只需要在现有基础上做确认、修正和补充，并提供正在使用的资料样本。

01

SOP 对齐

确认每条业务流的起点、终点、责任人、输入资料、输出结果和异常处理方式。

02

部门确认

由对应负责人判断节点是否准确，补充遗漏环节，并指出必须人工确认的关键位置。

03

资料收集

回收表格、系统截图、真实案例、话术模板、内容样本和数据报表，用于还原真实工作方式。

04

结果整理

根据确认结果整理正式 SOP、数据字段、人工把控点和后续深化沟通所需的事实依据。

SOP checkpoints

关键 SOP 节点拆分

以下每条流程都用于帮助对应部门确认现有工作方式。请重点查看：节点是否完整、顺序是否准确、责任人是否清楚、哪些环节必须人工确认。

01

Listing 上架流程

  1. 产品资料接收产品图、规格、卖点、成本、库存、目标市场。
  2. 关键词与竞品整理提取核心词、长尾词、竞品表达和平台限制。
  3. Listing 初稿生成标题、五点描述、详情页、多语言内容。
  4. 运营审核修改确认卖点、价格、合规词、平台规则。
  5. 上架发布录入平台、关联库存、记录版本。
  6. 效果回收曝光、点击、转化、退货和打回原因进入复盘。

AI 可辅助：关键词、标题、描述、多语言、违规词初筛。

人工确认：最终卖点、价格、合规风险、发布版本。

02

短视频制作流程

  1. 选题来源确认爆款参考、产品卖点、平台趋势、达人反馈。
  2. 脚本 / 分镜生成开头钩子、使用场景、卖点展开、CTA。
  3. 素材准备产品图、实拍素材、口播、字幕、品牌限制。
  4. AI 生成或剪辑批量生成多个版本，保留 Prompt 和素材来源。
  5. 成片审核检查产品呈现、合规、画面质量和平台适配。
  6. 发布与复盘记录发布账号、链接、互动、转化和可复用模板。

AI 可辅助：脚本、分镜、字幕、Prompt、多版本生成。

人工确认：产品表达、成片质量、是否可发布。

03

仓储 / 寄样 / BD 流程

  1. BD 确认达人确认达人账号、合作产品、报价和寄样必要性。
  2. 发起寄样申请提交产品 SKU、数量、收件信息、期望寄出时间。
  3. 仓库确认库存确认可用库存、样品状态、是否影响销售库存。
  4. 样品出库记录出库人、出库时间、物流单号和异常情况。
  5. 物流与签收跟进跟踪运输状态，确认达人是否收到样品。
  6. 发布链接回收回收视频链接、发布时间、互动数据和销售反馈。

AI 可辅助：库存提醒、寄样台账、物流状态、跟进提醒。

人工确认：是否寄样、寄哪个 SKU、异常处理、达人合作判断。

04

选品 / 采购 / 补货流程

  1. 候选产品收集来源、市场、竞品、目标售价和预估毛利。
  2. 产品评估供应稳定性、采购成本、物流限制、合规风险。
  3. 采购确认供应商、起订量、交期、付款方式和质检要求。
  4. 入库同步SKU、批次、数量、成本和可售状态进入库存表。
  5. 销售 / 寄样消耗反馈结合销售、达人寄样、退货和库存周转判断补货。
  6. 补货决策确认补货数量、时间、预算和优先级。

AI 可辅助：产品资料整理、风险提示、补货预测初稿。

人工确认：供应商选择、采购价格、补货预算。

05

达人开发 / 合作流程

  1. 达人池收集平台、账号、粉丝量、国家、类目、过往表现。
  2. 达人匹配根据产品、价格带、国家和内容风格筛选。
  3. 邀约沟通发送合作话术，记录报价、档期、寄样需求。
  4. 合作确认确认合作形式、产品、发布时间和交付要求。
  5. 寄样联动与仓库同步寄样申请、物流和签收状态。
  6. 效果沉淀发布链接、播放、互动、转化和合作评价进入达人库。

AI 可辅助：达人筛选、邀约话术、跟进提醒、效果归类。

人工确认：合作价格、是否寄样、合作优先级。

06

数据复盘 / 反哺流程

  1. 数据采集Listing、视频、达人、销售、库存、退货、异常。
  2. 数据关联把结果对应到产品、SKU、内容版本、达人和渠道。
  3. 表现归因分析高转化、低转化、缺货、退货和内容问题。
  4. 结论沉淀沉淀为产品标签、内容模板、达人标签和风险清单。
  5. 动作分发同步给产品、采购、仓储、运营、内容和 BD。
  6. 下一轮优化进入选品、补货、Listing、视频和达人合作调整。

AI 可辅助：数据汇总、异常识别、复盘报告初稿。

人工确认：投入方向、停止测试、补货和资源分配。

Confirmation dimensions

本次需要确认的三类边界

Department questionnaire

各部门确认问卷

请各部门围绕自己负责的环节进行确认即可，不需要重新撰写完整流程。每个问题都尽量以“确认 / 修正 / 补充”为主。

通用确认

  • 这张图中属于本部门的节点是否准确？
  • 是否有遗漏的前置或后置环节？
  • 每个节点目前由谁负责、谁审核、谁接收结果？
  • 目前主要通过表格、系统还是群聊完成流转？

运营 / Listing

  • 从选品到上架，中间实际经过哪些确认步骤？
  • 上架前必须拿到哪些产品资料？哪些资料经常缺失？
  • 标题、关键词、五点描述和详情页分别由谁生成和审核？
  • 平台打回、违规词、差评处理等问题如何记录？

内容 / 视频

  • 视频选题、脚本、素材、Prompt、成片分别由谁负责？
  • AI 生成内容目前在哪些环节可用，哪些环节仍需人工判断？
  • 成片被打回通常是什么原因？是否有记录？
  • 表现好的脚本、素材和 Prompt 是否有沉淀方式？

BD / 达人

  • 达人从筛选、建联、报价、寄样到发布的真实流程是什么？
  • 达人表目前记录哪些字段，哪些字段最影响合作判断？
  • 邀约话术、信任背书、赔偿承诺等是否有固定模板？
  • 发布链接、播放互动、转化结果如何回收？

仓储 / 供应链

  • 库存、样品库存、锁定库存分别如何记录和更新？
  • BD 发起寄样后，仓库如何确认库存和出库？
  • 物流单号、签收状态、异常情况由谁回填？
  • 缺货、错发、延迟、退货等异常如何通知相关部门？

数据 / 复盘

  • 目前哪些数据可以回收到产品、视频、达人、Listing 或库存？
  • 复盘周期是什么，复盘结论由谁记录、谁执行？
  • 哪些数据现在依赖人工统计，容易遗漏或重复？
  • 复盘结果如何反哺选品、内容、BD 和补货？

Meeting material matrix

请各部门准备的资料

请各部门负责人提前准备已有资料、真实案例和当前使用的表格 / 系统截图。无需重新整理成正式文档，能说明实际工作方式即可。

老板 / 管理层

建议准备

  • 重点确认：当前最希望先提升的环节，例如内容产能、达人合作、上架效率、库存准确率、数据复盘。
  • 关键风险：不能出错的事项，例如价格错误、库存不准、平台违规、达人合作失控、样品成本失控。
  • 现有工具：目前使用的表格、ERP、平台后台、项目管理工具、聊天群和审批方式。
  • 验收指标：希望第一阶段看到的结果，例如节省人力、缩短周期、减少返工、数据可追踪。
可按这个格式准备：

最需要确认的流程：______；最不能出错：______；目前主要工具：______；希望补充说明：______。

产品 / 采购

建议准备

  • 产品资料表：SKU、产品名、规格、图片、卖点、目标国家、售价、成本、库存、供应商。
  • 供应商资料：供应商名称、联系方式、起订量、交期、采购价、历史问题。
  • 缺失案例：最近 2-3 个因为资料不全导致 Listing、视频或 BD 推进慢的例子。
  • 补货判断：目前如何决定补货，参考哪些销售、库存、达人寄样或采购周期数据。
产品资料模板：

SKU / 产品名 / 核心卖点 / 图片链接 / 成本 / 售价 / 供应商 / 交期 / 当前库存 / 适合国家 / 注意事项。

仓储负责人

建议准备

  • 库存表：SKU、可售库存、样品库存、锁定库存、更新时间、负责人。
  • 出入库记录：入库、销售出库、寄样出库、退货入库、异常调整记录。
  • 寄样记录：BD、达人、产品、数量、物流单号、寄出时间、签收状态。
  • 异常案例：缺货、错发、漏发、延迟、物流异常、库存不一致的真实案例。
寄样记录模板：

申请人 / 达人账号 / SKU / 数量 / 收件信息 / 库存确认 / 出库时间 / 物流单号 / 签收状态 / 异常说明。

Listing / 平台运营

建议准备

  • 历史 Listing：表现较好和表现较差的 Listing 各 2-3 条。
  • 关键词资料：核心关键词、竞品关键词、禁用词、平台规则。
  • 审核记录：平台打回、内部返工、违规风险、修改前后版本。
  • 上架流程：从收到产品资料到最终发布，中间经过哪些人确认。
Listing 样例模板：

SKU / 平台 / 标题 / 核心关键词 / 五点描述 / 审核人 / 打回原因 / 发布时间 / 效果数据。

内容 / 视频团队

建议准备

  • 爆款参考：近期参考的视频链接、账号、平台和适用产品。
  • 脚本模板：常用开头、场景、卖点表达、CTA、字幕格式。
  • Prompt / 成片：已经使用过的 AI Prompt、生成结果、可复用素材。
  • 返工案例：最近成片被打回或重做的原因，例如卖点不准、画面差、违规、素材缺失。
视频复盘模板：

产品 / 选题来源 / 脚本版本 / Prompt / 成片链接 / 审核意见 / 发布链接 / 数据表现 / 可复用点。

BD / 达人运营

建议准备

  • 达人表：账号、平台、国家、类目、粉丝量、报价、合作状态、历史效果。
  • 邀约话术：常用私信、邮件、报价沟通、寄样说明模板。
  • 合作记录：已合作达人、产品、寄样、发布时间、发布链接、效果数据。
  • 跟进问题：沟通慢、寄样慢、不发布、链接回收难、效果难归因等实际案例。
达人合作模板：

达人账号 / 平台 / 国家 / 类目 / 适合产品 / 报价 / 寄样状态 / 发布链接 / 效果 / 后续评级。

数据 / 复盘负责人

建议准备

  • 数据字段：目前能拿到哪些数据，分别来自平台、ERP、达人、广告或人工表格。
  • 复盘表：现有周报、月报、项目复盘、达人复盘、视频复盘样例。
  • 关联方式：数据能否对应到 SKU、Listing、视频、达人、库存和订单。
  • 历史结论：过去复盘后形成过哪些有效动作，例如补货、停投、换达人、改 Listing。
复盘模板：

产品 / 渠道 / 内容或达人 / 投入 / 曝光 / 点击 / 转化 / 库存影响 / 问题归因 / 下一步动作。

建议会议目标： 确认 SOP 主流程、修正错误节点、补充遗漏环节，并收集各部门现有表格、案例和数据样本。
