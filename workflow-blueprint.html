Principle

AI 和人的分工原则

我们不建议直接做全自动系统，而是先把重复劳动、资料整理和生成工作交给 AI，把风险判断和关键确认留给人。

AI 负责提效

整理资料、生成初稿、拆解爆款、批量改写、匹配推荐、检查提醒、复盘总结。

人负责把控

确认产品、价格、库存、供应商、合规表达、成片质量、达人合作和最终投放判断。

系统负责沉淀

把每次产品、生成、审核、寄样、发布、反馈都记录下来，形成下一轮优化依据。

AI scope

AI 可介入方向

  • 资料标准化、产品卖点提炼、标签生成
  • 竞品 / 爆款拆解、脚本模板沉淀
  • Listing 初稿、多语言、关键词辅助
  • 视频脚本、分镜、字幕、Prompt 批量生成
  • 达人匹配、邀约话术、跟进提醒
  • 库存预警、寄样记录、异常提醒
  • 数据复盘、表现总结、下一轮建议

Human control

人工必须把控

  • 产品卖点、价格、库存和合规限制
  • 供应商选择、采购价格、交期判断
  • 实际库存、发货、寄样和异常处理
  • Listing 最终审核和平台上架确认
  • 视频成片质量和是否可发布
  • 达人合作、报价、寄样确认
  • 是否继续投入、补货或调整方向

Data assets

数据沉淀结构

  • 产品库、素材库、供应商库
  • 库存库、出入库记录、寄样记录
  • 竞品库、爆款内容库、脚本 / Prompt 库
  • Listing 文案库、关键词库、审核记录库
  • 达人库、合作状态库、发布效果库
  • 销售反馈库、复盘结论库

Next meeting

下一步对齐会议准备清单

建议老板先组织各部门负责人开短会，请大家提前按各自模块准备资料和真实案例。后续现场访谈会围绕这些问题逐项确认，而不是从零描述流程。

老板 / 管理层

  1. 当前最影响效率、成本或转化的业务环节分别是什么。
  2. 哪些业务结果必须老板或主管最终确认。
  3. 哪些错误是绝对不能发生的，例如价格、库存、合规、达人合作、平台违规。
  4. 第一阶段更希望优先优化 Listing、内容生产、BD 分发、仓储寄样，还是全链路轻量打通。
  5. 如果项目有效，应该用哪些指标判断：产出数量、返工次数、人工耗时、发布效果、数据可追溯性。
  6. 现阶段有哪些系统、表格或工具是必须保留并对接的。

产品 / 采购

  1. 新品从发现、确认、采购到交给运营，中间经过哪些步骤。
  2. 产品资料现在包含哪些字段，哪些字段是后续 Listing 和视频必须使用的。
  3. 产品图片、视频、说明书、规格、价格、库存、卖点分别由谁维护。
  4. 供应商、采购成本、交期、补货周期现在如何记录。
  5. 哪些产品信息经常缺失，导致运营、内容、BD 或仓储反复沟通。
  6. 哪些产品有合规限制、禁用词、特殊国家或平台限制。

仓储负责人

  1. 产品入库、出库、库存变动现在由谁记录，记录在哪里。
  2. 库存数据多久更新一次，运营和 BD 如何知道当前可用库存。
  3. 达人寄样申请由谁发起，仓库如何确认是否可寄。
  4. 样品出库、物流单号、签收状态现在如何跟踪。
  5. 缺货、错发、延迟、退货等异常现在如何通知相关人员。
  6. 哪些仓储数据需要同步给产品、运营、BD 或老板做决策。

Listing / 平台运营

  1. 一条 Listing 从拿到产品资料到正式上架，中间经过哪些步骤。
  2. 标题、五点描述、详情页、关键词、多语言内容分别如何生成。
  3. 写 Listing 前必须拿到哪些产品、素材、关键词和竞品资料。
  4. 哪些平台规则、禁用词或合规要求最容易导致打回。
  5. Listing 由谁初审、谁终审，审核意见如何反馈。
  6. 历史 Listing、关键词、审核失败原因是否有沉淀和复用。

内容 / 视频团队

  1. 视频选题从哪里来，爆款参考视频由谁收集和筛选。
  2. 脚本、分镜、字幕、CTA、AI 视频 Prompt 现在分别由谁负责。
  3. 一条视频从选题到成片，完整流转步骤是什么。
  4. 制作视频前必须拿到哪些产品资料和素材。
  5. 脚本或成片最常见的返工原因是什么。
  6. 哪些内容必须由老板、主管或运营确认后才能发布。
  7. 表现好的脚本、Prompt、成片是否会沉淀为模板。

BD / 达人运营

  1. 达人从寻找、筛选、邀约、报价、寄样到发布，完整步骤是什么。
  2. 达人信息表现在记录哪些字段，例如国家、类目、粉丝、报价、联系方式、历史效果。
  3. 如何判断某个达人适合某个产品。
  4. 邀约话术、跟进话术、寄样通知是否有模板。
  5. BD 如何向仓库发起寄样，如何跟踪物流和达人签收。
  6. 达人发布链接、播放互动、转化结果是否能对应到产品和样品。

数据 / 复盘

  1. 发布后目前收集哪些数据：播放、互动、点击、订单、达人反馈、平台审核结果。
  2. 数据是否能对应到具体产品、视频、达人、Listing 和库存。
  3. 多久复盘一次，复盘由谁组织，结论记录在哪里。
  4. 表现好的内容、达人、产品是否会被标记原因。
  5. 表现差的内容是否会记录失败原因，例如素材、脚本、达人、库存、平台规则。
  6. 复盘结论如何进入下一轮选品、Listing、内容和 BD 分发。
建议会议目标： 确认主流程、修正错误节点、补充遗漏环节，并收敛第一阶段最值得优先改造的业务链路。
