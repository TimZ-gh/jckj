Alignment

对齐会议准备清单

默认只看“需准备材料”；展开后再看具体问题。这样老板转发给负责人时，压力更小。

老板 / 管理层优先级、风险、验收
关键风险优先链路验收指标
  1. 当前最影响效率、成本或转化的业务环节分别是什么。
  2. 哪些业务结果必须老板或主管最终确认。
  3. 哪些错误绝对不能发生，例如价格、库存、合规、达人合作、平台违规。
  4. 第一阶段更希望优先优化哪条链路。
  5. 项目有效应看哪些指标。
  6. 哪些系统、表格或工具必须保留并对接。
产品 / 采购资料、供应商、交期
产品资料供应商缺失字段
  1. 新品从发现、确认、采购到交给运营，中间经过哪些步骤。
  2. 产品资料包含哪些字段，哪些字段后续必须使用。
  3. 产品图片、视频、说明书、规格、价格、库存、卖点分别由谁维护。
  4. 供应商、采购成本、交期、补货周期如何记录。
  5. 哪些产品信息经常缺失。
  6. 哪些产品有合规限制、禁用词、特殊国家或平台限制。
仓储负责人库存、寄样、异常
库存表寄样记录异常记录
  1. 产品入库、出库、库存变动由谁记录，记录在哪里。
  2. 库存数据多久更新一次，运营和 BD 如何知道可用库存。
  3. 达人寄样申请由谁发起，仓库如何确认是否可寄。
  4. 样品出库、物流单号、签收状态如何跟踪。
  5. 缺货、错发、延迟、退货等异常如何通知相关人员。
  6. 哪些仓储数据需要同步给产品、运营、BD 或老板。
Listing / 平台运营上架、关键词、审核
上架步骤关键词打回原因
  1. 一条 Listing 从产品资料到上架，中间经过哪些步骤。
  2. 标题、五点描述、详情页、关键词、多语言内容如何生成。
  3. 写 Listing 前必须拿到哪些资料。
  4. 哪些平台规则或禁用词最容易导致打回。
  5. Listing 由谁初审、谁终审。
  6. 历史 Listing、关键词、审核失败原因是否沉淀。
内容 / 视频团队选题、脚本、成片
爆款样本脚本模板返工原因
  1. 视频选题从哪里来，爆款参考由谁收集和筛选。
  2. 脚本、分镜、字幕、CTA、AI Prompt 分别由谁负责。
  3. 一条视频从选题到成片，完整流转步骤是什么。
  4. 制作前必须拿到哪些产品资料和素材。
  5. 脚本或成片最常见返工原因是什么。
  6. 哪些内容必须确认后才能发布。
  7. 表现好的脚本、Prompt、成片是否沉淀为模板。
BD / 达人运营达人、寄样、发布
达人表邀约话术发布链接
  1. 达人从寻找、筛选、邀约、报价、寄样到发布的完整步骤。
  2. 达人信息表记录哪些字段。
  3. 如何判断达人适合某个产品。
  4. 邀约、跟进、寄样通知是否有模板。
  5. BD 如何向仓库发起寄样并跟踪物流。
  6. 达人发布和转化结果是否能对应到产品和样品。
数据 / 复盘归因、结论、反哺
数据字段复盘表结论记录
  1. 发布后收集哪些数据。
  2. 数据能否对应到具体产品、视频、达人、Listing 和库存。
  3. 多久复盘一次，谁组织，结论记录在哪里。
  4. 表现好的内容、达人、产品是否标记原因。
  5. 表现差的内容是否记录失败原因。
  6. 复盘结论如何进入下一轮选品、Listing、内容和 BD。
建议会议目标： 确认主流程、修正错误节点、补充遗漏环节，并收敛第一阶段最值得优先改造的业务链路。
